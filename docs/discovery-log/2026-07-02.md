## Source implementation: Running, Ballard

- ✅ Added: **Fleet Feet Seattle-Ballard** — ICS (public Google Calendar export) — 14 upcoming events confirmed live via `ONLY_SOURCE` build — running specialty store at 5404 22nd Ave NW, Ballard; the store's `/events` page embeds Google Calendar event links whose base64 `eid` decodes to a public calendar id, exposing a standard `calendar.google.com/calendar/ical/.../public/basic.ics` feed with an indefinite weekly Tuesday group run plus dated brand demo-run overrides — `sources/external/fleet-feet-seattle-ballard.yaml`, `docs/source-candidates/fleet-feet-seattle-ballard.md`
- 🔧 Fix: while verifying this source, found that `parseExternalCalendarEvents` (`lib/tag_aggregator.ts`) didn't account for `RECURRENCE-ID` override VEVENTs when expanding an RRULE — a Google-Calendar-style "this occurrence is actually something else" override was emitted alongside the generic recurring instance it replaces, producing a duplicate event on the overridden date. Fixed the RRULE expansion to skip occurrences with a matching override and added a regression test.
